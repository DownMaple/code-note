Vite 8 升级指南 ​

2026 年 3 月 12 日，Vite 8.0 正式发布，这是 Vite 自 2.0 以来最重大的架构变革——用基于 Rust 的 Rolldown 统一打包工具替代了此前 esbuild + Rollup 的双引擎架构。

架构变化：为什么换掉 esbuild + Rollup？ ​

之前的痛点 ​

Vite 7 及之前版本使用两套独立系统：

  • 开发阶段：esbuild（极快的转换和依赖预构建）
  • 生产构建：Rollup（成熟的打包和 Tree Shaking）

这种分割导致了固有摩擦：两条转换管道、两个插件系统，开发和生产之间行为不一致。

Vite 8 的方案 ​

Vite 8 形成了完整的端到端 Rust 工具链：

角色Vite 7Vite 8
构建工具ViteVite
打包工具esbuild（开发）+ Rollup（生产）Rolldown（统一）
JS/TS 编译esbuildOxc
CSS 压缩esbuildLightning CSS
JS 压缩esbuildOxc Minifier

三者（Vite、Rolldown、Oxc）均由 VoidZero 团队维护，行为一致性得到保证。

新特性详解 ​

内置 tsconfig paths 支持 ​

之前需要安装 vite-tsconfig-paths 插件，现在内置支持：

ts
// vite.config.ts
import { defineConfig } from 'vite'

export default defineConfig({
  resolve: {
    tsconfigPaths: true,
  },
})

配置后 Vite 会自动读取 tsconfig.json 中的 paths 配置进行路径解析，无需额外插件。

TIP

该选项默认为 false，因为有轻微性能开销。如果你的项目使用了 tsconfig paths，建议手动开启。

Oxc 替代 esbuild 的转换配置 ​

Vite 8 使用 Oxc 替代 esbuild 进行 JS/TS 转换，配置项也随之变化：

ts
// vite.config.ts — Vite 7（旧）
export default defineConfig({
  esbuild: {
    jsx: 'automatic',
    jsxFactory: 'h',
    jsxFragment: 'Fragment',
  },
})

// vite.config.ts — Vite 8（新）
export default defineConfig({
  oxc: {
    jsx: {
      runtime: 'automatic',   // 'classic' | 'automatic'
      pragma: 'h',            // JSX 工厂函数
      pragmaFrag: 'Fragment', // Fragment 工厂函数
    },
  },
})

Vite 8 还内置支持了 TypeScript 的 emitDecoratorMetadata，无需外部插件。

集成 Devtools（实验性） ​

直接在开发服务器中集成调试工具，提供模块图检查和转换管道实时调试：

bash
npm install @vitejs/devtools
ts
// vite.config.ts
export default defineConfig({
  devtools: true,
})

浏览器控制台转发 ​

将浏览器的 console 输出和运行时错误转发到 CLI 终端，特别适合与 AI 编码助手（Cursor、Copilot 等）配合使用：

ts
// vite.config.ts
export default defineConfig({
  server: {
    forwardConsole: {
      unhandledErrors: true,
      logLevels: ['warn', 'error'],
    },
    // 简写：forwardConsole: true
  },
})

INFO

当 Vite 检测到 AI 编码助手环境时，默认会自动开启此功能。

默认使用 Oxc 压缩 ​

ts
// vite.config.ts
export default defineConfig({
  build: {
    minify: 'oxc',      // 默认值（新）
    // minify: 'terser', // 仍然支持
    // minify: 'esbuild',// 已废弃，可用于回退
  },
})

Lightning CSS 作为默认 CSS 压缩 ​

ts
// vite.config.ts
export default defineConfig({
  build: {
    cssMinify: 'lightningcss', // 默认值（新）
    // cssMinify: 'esbuild',   // 回退选项
  },
})

构建目标更新 ​

ts
// vite.config.ts
export default defineConfig({
  build: {
    target: 'baseline-widely-available', // 新默认值
  },
})

对应的最低浏览器版本：

Rolldown 配置（替代 rollupOptions） ​

ts
// vite.config.ts
export default defineConfig({
  build: {
    // Vite 7（旧）
    // rollupOptions: { ... }

    // Vite 8（新）
    rolldownOptions: {
      // Rolldown 配置
    },
  },
})

许可证文件自动生成 ​

ts
// vite.config.ts
export default defineConfig({
  build: {
    license: true, // 自动生成 .vite/license.md
  },
})

Lightning CSS 作为 CSS 转换器（实验性） ​

除了默认的 CSS 压缩外，还可以用 Lightning CSS 完全替代 PostCSS 进行 CSS 转换：

ts
// vite.config.ts
export default defineConfig({
  css: {
    transformer: 'lightningcss', // 'postcss'（默认） | 'lightningcss'
    lightningcss: {
      targets: { /* 浏览器目标 */ },
      drafts: { /* CSS 草案特性 */ },
      cssModules: { /* CSS Modules 配置 */ },
    },
  },
})

Module Federation 支持 ​

Rolldown 统一打包器解锁了 Module Federation 的原生支持，多个独立构建的应用可以在运行时共享模块，适用于微前端架构场景。

WebAssembly SSR 支持 ​

.wasm?init 导入现在在 SSR 上下文中也能正常工作，之前仅限客户端使用：

ts
import init from './example.wasm?init'

const instance = await init()
instance.exports.greet()

Full Bundle Mode（实验性） ​

开发阶段也进行打包（类似生产模式），而非以往的按需加载：

ts
// vite.config.ts
export default defineConfig({
  dev: {
    // 开发环境启用完整打包模式
    bundler: true,
  },
})

效果显著：

  • 开发服务器启动速度提升约 3 倍
  • 完整重载快 40%
  • 网络请求减少 10 倍

@vitejs/plugin-react v6 ​

使用 Oxc 替代 Babel 处理 React Refresh 转换，Babel 不再是必要依赖：

bash
npm install @vitejs/plugin-react@6

安装更快、HMR 更快。

迁移指南（Vite 7 → Vite 8） ​

环境要求 ​

  • Node.js 20.19+ 或 22.12+（需支持无标志 require(esm)）

升级方式 ​

方式一：直接升级

bash
npm install vite@8

方式二：渐进式迁移（推荐大型项目）

先用 rolldown-vite 包测试兼容性：

bash
npm install rolldown-vite

确认无问题后，再切换到 vite@8。

Breaking Changes ​

1. 配置项变更 ​

旧配置新配置状态
build.rollupOptionsbuild.rolldownOptions废弃（自动转换）
worker.rollupOptionsworker.rolldownOptions废弃（自动转换）
build.commonjsOptions-废弃（无操作）
optimizeDeps.esbuildOptionsrolldownOptions废弃
esbuildoxc废弃（自动映射）

TIP

Vite 8 提供了兼容层，会自动将旧配置转换为新配置。但建议在迁移后手动更新配置文件。

2. JS 压缩变更 ​

  • Oxc Minifier 替代 esbuild 成为默认压缩器
  • 不支持 mangleProps 等属性名混淆
  • 如需回退，可设置 build.minify: 'esbuild'（已废弃但可用）

3. CSS 压缩变更 ​

  • Lightning CSS 成为默认 CSS 压缩工具
  • CSS 包体积可能略有增加

4. CommonJS 互操作一致性变更 ​

default 导入行为更加一致：

ts
// .mjs/.mts 文件或 "type": "module" 的包，
// 未设置 __esModule 时：导入 module.exports
import foo from 'cjs-pkg'

// 其他情况：导入 module.exports.default

如需保持旧行为，临时回退：

ts
export default defineConfig({
  legacy: {
    inconsistentCjsInterop: true,
  },
})

5. require 调用变更 ​

外部模块的 require 调用现在保留原样（不转换为 import）。如需转换，使用内置插件：

ts
import { defineConfig, esmExternalRequirePlugin } from 'vite'

export default defineConfig({
  plugins: [
    esmExternalRequirePlugin({
      external: ['react', 'vue', /^node:/],
    }),
  ],
})

6. 装饰器支持（部分） ​

Oxc 暂不支持原生装饰器降级。如果你的项目使用了装饰器，需要 Babel 或 SWC：

ts
import { defineConfig } from 'vite'
import babel from '@rolldown/plugin-babel'

function decoratorPreset(options) {
  return {
    preset: () => ({
      plugins: [['@babel/plugin-proposal-decorators', options]],
    }),
    rolldown: {
      filter: { code: '@' },
    },
  }
}

export default defineConfig({
  plugins: [
    babel({
      presets: [decoratorPreset({ version: '2023-11' })],
    }),
  ],
})

7. 插件作者注意 ​

load/transform 钩子返回值需要添加 moduleType: 'js'：

ts
const plugin = {
  name: 'txt-loader',
  load(id) {
    if (id.endsWith('.txt')) {
      const content = fs.readFileSync(id, 'utf-8')
      return {
        code: `export default ${JSON.stringify(content)}`,
        moduleType: 'js', 
      }
    }
  },
}

8. build() API 变更 ​

build() 函数现在抛出 BundleError 而非原始错误：

ts
try {
  await build()
} catch (e) {
  if (e.errors) {
    for (const error of e.errors) {
      console.log(error.code)
    }
  }
}

9. 其他限制 ​

  • 暂不支持 Extglobs
  • TypeScript 命名空间仅部分支持
  • 不支持转换至 ES5 及以下
  • 不支持输出格式 'system' 和 'amd'
  • 安装体积增加约 15 MB（lightningcss ~10MB + Rolldown ~5MB）

一个完整的 Vite 8 配置示例 ​

ts
// vite.config.ts
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

export default defineConfig({
  plugins: [vue()],

  resolve: {
    tsconfigPaths: true,
  },

  oxc: {
    jsx: {
      runtime: 'automatic',
    },
  },

  server: {
    forwardConsole: true,
  },

  build: {
    target: 'baseline-widely-available',
    minify: 'oxc',
    cssMinify: 'lightningcss',
    rolldownOptions: {
      output: {
        // Rolldown 输出配置
      },
    },
    license: true,
  },

  // devtools: true, // 按需开启
})

生态系统兼容性 ​

已验证兼容的框架：SvelteKit、React Router、Storybook。

对于 Astro、Nuxt、Vitest 等框架用户，可能需要在 package.json 中覆盖 Vite 依赖：

json
{
  "overrides": {
    "vite": "^8.0.0"
  }
}

Vite 8 同时上线了插件注册中心 registry.vite.dev，可以搜索 Vite、Rolldown 和 Rollup 的插件。
