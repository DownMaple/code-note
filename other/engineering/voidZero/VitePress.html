VitePress 使用指南 ​

VitePress 是基于 Vite 和 Vue 的静态站点生成器（SSG），主要面向文档类网站。它是 VuePress 的精神继承者，更轻量、更快。

核心特性 ​

  • 基于 Vite：开发服务器即时热更新，构建速度极快
  • 文件路由：Markdown 文件自动映射为页面路由
  • Markdown 扩展：代码高亮（Shiki）、自定义容器、代码组等
  • Vue 集成：可在 Markdown 中直接使用 Vue 组件和模板语法
  • 默认主题：开箱即用，支持暗黑模式、搜索、导航等

与 VuePress 的区别 ​

维度VitePressVuePress
构建工具ViteWebpack
开发体验即时热更新，毫秒级启动较慢的冷启动
Vue 版本Vue 3VuePress 2 用 Vue 3
代码高亮ShikiPrism
定位更轻量，面向文档插件生态更成熟

快速开始 ​

前置条件 ​

  • Node.js 18+（推荐 20+）
  • 支持 Markdown 语法的编辑器（推荐 VS Code）

安装 ​

bash
# npm
npm add -D vitepress

# pnpm
pnpm add -D vitepress

# yarn（需额外安装 vue）
yarn add -D vitepress vue

# bun
bun add -D vitepress

初始化 ​

bash
npx vitepress init

向导会询问：配置文件位置（默认 ./docs）、网站标题、描述、主题选择、是否启用 TypeScript。

项目结构 ​

.
├─ docs
│  ├─ .vitepress
│  │  ├─ config.mts        # 站点配置文件
│  │  ├─ cache/            # 缓存（应加入 .gitignore）
│  │  ├─ dist/             # 构建输出
│  │  └─ theme/            # 自定义主题目录（可选）
│  │     └─ index.ts
│  ├─ index.md             # 首页
│  ├─ guide/
│  │  └─ getting-started.md
│  └─ public/              # 静态资源（原样拷贝到输出目录）
└─ package.json

开发与构建 ​

json
{
  "scripts": {
    "docs:dev": "vitepress dev docs",
    "docs:build": "vitepress build docs",
    "docs:preview": "vitepress preview docs"
  }
}
bash
# 开发 — 默认 http://localhost:5173
npm run docs:dev

# 构建 — 输出到 docs/.vitepress/dist
npm run docs:build

# 预览构建结果 — http://localhost:4173
npm run docs:preview

站点配置 ​

基本配置 ​

ts
// docs/.vitepress/config.mts
import { defineConfig } from 'vitepress'

export default defineConfig({
  lang: 'zh-CN',
  title: 'My Docs',
  description: '项目文档',
  base: '/',                  // 部署子路径，如 '/blog/'
  cleanUrls: true,            // URL 中去除 .html
  lastUpdated: true,          // 显示 Git 最后更新时间

  head: [
    ['link', { rel: 'icon', href: '/favicon.ico' }],
    ['meta', { name: 'keywords', content: 'vitepress,docs' }],
  ],

  themeConfig: {
    // 主题配置（见下文）
  },
})

Markdown 扩展 ​

VitePress 对标准 Markdown 做了很多增强，这些是写文档时最常用的功能。

代码块语法高亮 ​

使用 Shiki 引擎，在反引号后指定语言：

md
```ts
const msg: string = 'Hello VitePress'
console.log(msg)
```

行高亮 ​

用花括号指定行号：

md
```ts{1,3-4}
const a = 1       // 高亮
const b = 2
const c = 3       // 高亮
const d = 4       // 高亮
```

也可以用行内注释：

ts
const a = 1
const b = 2

代码差异（Diff） ​

ts
export default {
  data() {
    return {
      msg: 'Removed',  
      msg: 'Added',    
    }
  },
}

代码聚焦 ​

ts
export default {
  data() {
    return {
      msg: 'Focused!', 
    }
  },
}

错误和警告标记 ​

ts
console.log('Error here')   
console.log('Warning here')

行号 ​

全局开启：

ts
export default defineConfig({
  markdown: {
    lineNumbers: true,
  },
})

单个代码块覆盖：在语言后加 :line-numbers 或 :no-line-numbers。

自定义起始行号：:line-numbers=5。

代码组 ​

多个代码块以标签页形式展示：

md
::: code-group

```js [config.js]
const config = { /* ... */ }
export default config
```

```ts [config.ts]
const config: UserConfig = { /* ... */ }
export default config
```

:::

自定义容器 ​

md

这是一条信息。
:::

::: tip
这是一个提示。
:::

::: warning
这是一个警告。
:::

::: danger
这是一个危险警告。
:::

导入代码片段 ​

从文件中导入代码：

md
<<< @/snippets/example.ts

<!-- 指定高亮行 -->
<<< @/snippets/example.ts{2}

<!-- 导入文件中的指定区域 -->
<<< @/snippets/example.ts#snippet

在源文件中定义区域：

ts
// #region snippet
function foo() {
  console.log('hello')
}
// #endregion snippet

在 Markdown 中使用 Vue ​

VitePress 允许在 Markdown 中直接使用 Vue 的能力。

使用 Vue 模板语法 ​

md
当前文档数据：{{ $frontmatter.title }}

使用 <script setup> ​

md
# 我的页面

<script setup>
import { ref } from 'vue'
const count = ref(0)
</script>

当前计数：{{ count }}

<button @click="count++">+1</button>

使用自定义组件 ​

先注册组件（在主题文件中）：

ts
// .vitepress/theme/index.ts
import DefaultTheme from 'vitepress/theme'
import MyComponent from './components/MyComponent.vue'

export default {
  extends: DefaultTheme,
  enhanceApp({ app }) {
    app.component('MyComponent', MyComponent)
  },
}

然后在 Markdown 中直接使用：

md
# 我的页面

<MyComponent />

布局插槽 ​

默认主题提供了丰富的插槽，可以在不同位置注入自定义内容：

vue
<!-- .vitepress/theme/MyLayout.vue -->
<script setup>
import DefaultTheme from 'vitepress/theme'
const { Layout } = DefaultTheme
</script>

<template>
  <Layout>
    <template #doc-top>
      文档顶部自定义内容
    </template>
    <template #aside-outline-before>
      侧边栏顶部自定义内容
    </template>
  </Layout>
</template>

常用插槽：

区域插槽名
文档doc-top、doc-bottom、doc-before、doc-after
侧边栏sidebar-nav-before、sidebar-nav-after
主页home-hero-before、home-hero-after、home-features-before、home-features-after
全局layout-top、layout-bottom

Vite 配置透传 ​

VitePress 配置文件支持直接传入 Vite 和 Vue 的配置：

ts
export default defineConfig({
  vite: {
    // 任意 Vite 配置
    resolve: {
      alias: { /* ... */ },
    },
    plugins: [ /* ... */ ],
  },
  vue: {
    // @vitejs/plugin-vue 选项
  },
})

动态路由 ​

适用于从数据生成多个页面的场景。

创建 packages/[pkg].md 和 packages/[pkg].paths.js：

js
// packages/[pkg].paths.js
export default {
  paths() {
    return [
      { params: { pkg: 'vue' }, content: '...' },
      { params: { pkg: 'react' } },
    ]
  },
}

页面中通过 useData() 获取参数：

vue
<script setup>
import { useData } from 'vitepress'
const { params } = useData()
</script>

当前包：{{ params.pkg }}

构建钩子 ​

VitePress 提供了多个构建钩子，可以在构建过程中处理数据：

ts
export default defineConfig({
  // 转换每个页面的 <head>
  async transformHead(context) {
    // 返回额外的 head 标签
  },

  // 转换页面数据
  async transformPageData(pageData) {
    pageData.contributors = await getContributors(pageData.relativePath)
  },

  // 构建完成后
  async buildEnd(siteConfig) {
    // 生成额外文件（如 sitemap）
  },
})

国际化（i18n） ​

VitePress 内置多语言支持，通过目录结构和配置实现：

目录结构 ​

docs/
├─ en/
│  ├─ guide/
│  │  └─ getting-started.md
│  └─ index.md
├─ zh/
│  ├─ guide/
│  │  └─ getting-started.md
│  └─ index.md
└─ .vitepress/
   └─ config.mts

配置 ​

ts
export default defineConfig({
  locales: {
    root: {
      label: '简体中文',
      lang: 'zh-CN',
      themeConfig: {
        nav: [{ text: '指南', link: '/guide/' }],
        sidebar: { /* 中文侧边栏 */ },
      },
    },
    en: {
      label: 'English',
      lang: 'en-US',
      link: '/en/',
      themeConfig: {
        nav: [{ text: 'Guide', link: '/en/guide/' }],
        sidebar: { /* 英文侧边栏 */ },
      },
    },
  },
})

配置后导航栏会自动出现语言切换下拉菜单。root 表示默认语言，对应根目录下的文件。

部署 ​

GitHub Pages ​

创建 .github/workflows/deploy.yml：

yaml
name: Deploy VitePress
on:
  push:
    branches: [main]

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0        # lastUpdated 需要完整 git 历史
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm
      - run: npm ci
      - run: npm run docs:build
      - uses: actions/upload-pages-artifact@v3
        with:
          path: docs/.vitepress/dist

  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    needs: build
    runs-on: ubuntu-latest
    steps:
      - uses: actions/deploy-pages@v4
        id: deployment

如果部署到 https://<user>.github.io/<repo>/，需设置 base: '/<repo>/'。

Netlify ​

  • 构建命令：npm run docs:build
  • 输出目录：docs/.vitepress/dist
  • Node 版本：20+

Vercel ​

vercel.json：

json
{
  "headers": [
    {
      "source": "/assets/(.*)",
      "headers": [
        {
          "key": "Cache-Control",
          "value": "max-age=31536000, immutable"
        }
      ]
    }
  ]
}

TIP

生产构建的静态资源使用哈希文件名，建议对 /assets/* 配置强缓存。注意不要启用 HTML 自动压缩，否则会移除 Vue 需要的注释节点，导致水合错误。
